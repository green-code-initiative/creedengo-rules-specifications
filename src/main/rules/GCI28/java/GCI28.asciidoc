== Why is this an issue?

`try...catch` for file reading wastes CPU/RAM on exception overhead.
Use explicit checks instead.

== Examples

=== Noncompliant

[source,java,data-diff-id="1",data-diff-type="noncompliant"]
----
public void readPreferences(String filename) {
    //...
    InputStream in = null;
    try {
        in = new FileInputStream(filename);
    } catch (FileNotFoundException e) {
        logger.log(e);
    }
    in.read(...);
    //...
}
----

=== Compliant

[source,java,data-diff-id="2",data-diff-type="compliant"]
----
public void readPreferences(String filename) throws IllegalArgumentException, FileNotFoundException, IOException {
    if (filename == null) {
        throw new IllegalArgumentException ("filename is null");
    }
    //...
    InputStream in = new FileInputStream(filename);
    //...
}
----
