== Why is this an issue?

The form `$i++` creates a temporary variable whereas `++$i` does not. It saves CPU cycles.

== Examples

=== Noncompliant

[source,java,data-diff-id="1",data-diff-type="noncompliant"]
----
i++ //Noncompliant
----

=== Compliant

[source,java,data-diff-id="2",data-diff-type="compliant"]
----
++i
----

=== Exception
In some cases, it may be intentional to allow the use of `i++`, even if it is probably not far from a code smell.
Examples:

[source,java,data-diff-id="3",data-diff-type="compliant"]
----
void bar(int value) {
    // ...
}

int foo() {
    int i = 0;
    bar(i++);
    return i;
}
----
or
[source,java,data-diff-id="1",data-diff-type="compliant"]
----
private int i = 0;
int foo() {
    return this.i++;
}
---
