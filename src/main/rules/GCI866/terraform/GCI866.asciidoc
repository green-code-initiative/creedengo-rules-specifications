== Why is this an issue ?

Two Terraform anti-patterns inflate the operational + embodied carbon footprint
of an infrastructure without any functional gain in many projects:
- **Reflex multi-region** : deploying the same workload in
    several regions of the same cloud provider — through aliased
    <code>provider</code> blocks, a <code>for_each = var.regions</code>
    loop, or an explicit cross-region replication block — multiplies
    both the embodied (M) and operational (E·I) terms of the
    <em>Software Carbon Intensity</em> (SCI) equation by roughly the
    number of replicated regions, even when no RPO/RTO constraint
    justifies it.
- **Always-on compute** : provisioning long-running VMs
      (<code>aws_instance</code>, <code>azurerm_linux_virtual_machine</code>,
      <code>google_compute_instance</code>, etc.), managed databases at
      provisioned tier, or non-zero-min autoscaling groups, when a fully
      managed serverless equivalent (AWS Lambda / Fargate / Aurora
      Serverless v2, Azure Functions / Container Apps, GCP Cloud Run /
      Cloud Functions / GKE Autopilot) would enable scale-to-zero and
      eliminate idle-time energy cost.

== How to fix it ?

This rule does **not** forbid either pattern — it asks the
author to <em>document the choice</em> or pick a more efficient
alternative:
- If multi-region is required (RPO/RTO, data sovereignty), add a
    comment <code># eco-design:multiregion=&lt;reason&gt;</code> on the
    provider/resource block to silence the warning.
- If the always-on resource is mandatory (legacy software, OS-level
    agent, stateful workload), add
    <code># eco-design:serverless=&lt;reason&gt;</code> on the block.
- Otherwise, replace the resource by its serverless equivalent or
    consolidate the workload in a single region.

== Examples

=== Noncompliant

[source,terraform,data-diff-id="1",data-diff-type="noncompliant"]
----
# Reflex multi-region — no documented RPO/RTO.
provider "aws"               { region = "eu-west-3" }
provider "aws" { alias = "dr" region = "us-east-1" }

# Always-on VM where a Lambda would fit a bursty 5-min/hour load.
resource "aws_instance" "cron_runner" {
  instance_type = "t3.medium"
}
----

=== Compliant

[source,terraform,data-diff-id="3",data-diff-type="compliant"]
----
# eco-design:serverless=cold-start unacceptable for trading engine
resource "aws_instance" "trading_engine" {
  instance_type = "c7g.large"
}

# Single-region — no DR requirement.
provider "aws" {
  region = "eu-west-3"
}
----
