== Why is this an issue?

When the final size of a string or buffer is known in advance, initializing it with the appropriate capacity avoids dynamic resizing during appends.
Dynamic resizing consumes additional CPU cycles (for memory allocation and copying) and RAM, leading to unnecessary energy consumption.

== Examples

=== Noncompliant

[source,java,data-diff-id="1",data-diff-type="noncompliant"]
----
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100; i++) {
    sb.append(...);
}
----

=== Compliant

[source,java,data-diff-id="1",data-diff-type="compliant"]
----
StringBuilder sb = new StringBuilder(100);
for (int i = 0; i < 100; i++) {
    sb.append(...);
}
----
