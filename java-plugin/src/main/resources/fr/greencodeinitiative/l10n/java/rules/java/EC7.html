Using persistence Entity in the return of a Rest controler is not energy efficient. This use induces a multiples unnecessary SQL calls, multiple unused fields serialization, and thus unnecessary calculations by the CPU, RAM usage and network usage.

Noncompliant Code Example

@RestController
public class BookController {
    private final BookService bookService;

    public BookController(BookService bookstoreService) {
        this.bookService = bookstoreService;
    }

    @RequestMapping("books")
    public List<Book> findBooks() {
        return bookService.findAll();
    }
}

Compliant Solution

@RestController
public class BookController {
    private final BookService bookService;

    public BookController(BookService bookstoreService) {
        this.bookService = bookstoreService;
    }

    @RequestMapping("books")
    public List<BookDTO> findBooks() {
        return BookMapper.from(bookService.findAll());
    }
}
